网络安全先导

一、信息系统安全三要素

（一）信息安全

保密性 Confidential、完整性 Integrity、可用性 Availability
抗依赖性、可控性、真实性、时效性、合规性、公平性、可靠性、隐私性

（二）网络空间安全

包括了国家安全，城市安全、经济安全、社会安全、生产安全、人生安全..
网络空间和海陆空天并称 五大空间 4

（三）网络空间关注点

-信息系统——网络空间域，物理空间域，社会空间域 -技术——法律、政策、技术、管理、产业、教育
信息系统生命周期——时时刻刻

（四）网络空间安全管理流程

  1. 确定网络信息安全管理对象
  2. 评估网络信息安全管理对象价值
  3. 识别网络信息安全管理对象威胁
  4. 识别网络信息安全管理对象脆弱性
  5. 确定网络信息安全管理对象风险级别
  6. 制定网络信息安全防范体系、防范措施
  7. 十四号和落实网络信息安全保障
  8. 运行/维护网络信息安全设备、配置

二、网络安全常用术语

第一部分

  1. 漏洞 vulnerability
    硬件 软件 协议等等中 存在的安全缺陷
    提交平台：360 补天 CNNVD。

  2. POC （Proof of Concept）
    能证明漏洞存在的代码
    证明：dnslog 平台

  3. EXP（exploit）利用
    执行了这段利用代码可以达到攻击的目的
    msf

  4. payload 攻击载荷

  • SQL 注入
  • XSS 注入
  • log4i

  1. 0day 使用量大 影响大的产品漏洞、尚未公开和修复的漏洞

  2. 1day 漏洞的 POC 和 EXP 已经被公开了，但是很多人来不及修复，这个叫做 1day 漏洞

  3. Nday 漏洞 发布官方补丁的漏洞 且发布时间较长

  4. 漏扫基于数据库对漏洞进行自动化扫描

  5. 补丁 patch
    漏洞的修复程序（覆盖）

  6. 渗透 penetration
    黑客入侵网站/计算级系统，获取到控制

  7. 渗透测试 penetration test
    安全测试，此过程中不可以影响系统运行和数据完整

  8. 木马 Trojan horse
    隐藏在计算机中的恶意程序
    不自动运行 需要调用

  9. 病毒 Virus
    恶意程序或代码
    （安全性 Linux>mac>windows）

  10. 杀毒软件–免杀（绕过杀毒软件）

第二部分

  1. 肉鸡
    已经被黑客获得控制权限的机器，可能是个人电脑 或者 企业政府的服务器

  2. 抓鸡
    利用出现概率非常高的漏洞，使用自动化方式获取肉鸡

  3. 跳板机
    黑客为了防止被追溯和识别身份，一般都不会用自己的电脑发起攻击，而是利用获取的肉鸡攻击其他目标，这个肉鸡就充当一个跳板的角色

  4. DDoS 分布式拒绝服务攻击
    发起大量恶意访问请求，导致正常用户无法访问

  5. 后门 backdoor
    黑客为了对主机进行长期的控制，在机器上种植的一段程序或留下的一个“入口”
    （多以木马形式存在）

  6. 中间人攻击 MITM 攻击
    运行中间服务器，拦截并篡改数据

第三部分

  1. webshell
    shell 是一种命令执行工具，可以和计算机系统进行控制
    webshell 就是 asp、php、jsp 之外的 web 代码文件，通过这些代码文件可以执行任意的命令，对计算机做任意的操作
  • 小马（只有文件上传功能）
  • 一句话木马（核心内容只有一句代码）godzilla/behinder/ant sword 网站管理工具
    ——> heckbar 插件输入 Post 请求
  • 大马 集合相关工具

  1. Getshell
    获得命令执行环境的操作
    Redis 的持久化功能，MySQL 的写文件功能
    MySQL 的日志功能功能、上传功能、数据备份功能、编辑器

  2. 提权 权限提升 PRivilege EScalation
    普通用户得到管理员权限
    linux：root

  3. 拿站
    指得到网站的最高权限，即得到后台和管理员名字和密码

  4. 拖库（脱裤）
    网站入侵后，渗透者把全部数据都导出到了数据文件中
    这些数据文件叫做 裤子（违法行为）

  5. 撞库
    用获得的裤子去批量登录其他网站

  6. 旁站入侵
    入侵同一服务器的其他网站（同一公司的许多下属网站/虚拟主机 大家共享了一个 ip）

  7. 横向移动（内网中）
    攻击者入侵一台服务器后，基于内部网络，继续入侵该网段的其他机器

第四部分

  1. 代理 Proxy
    帮我们发起请求的一台服务器
  • 突破限制 连接服务器
  • 访问内部资源
  • 隐藏真实 ip
  1. VPN Virtual Private Network
  • 代理
  • 加密通信
  • 办公/在家连接 公司内网

  1. 蜜罐 Honeypot
    （免费的微步 HFish）
    吸引攻击者攻击的伪装系统，用来实现溯源和反制
    盖以诱敌！

  2. 沙箱 Sandbox
    沙箱是一种按照安全策略限制程序行为的 执行环境，就算有恶意代码，
    也只能影响沙箱环境而不会影响到操作系统

  3. 靶场 模拟有漏洞的环境
    网站/容器/操作系统

  • web 综合靶场 DVWA pikachu bwapp
  • web 专用靶场 sqli-labs upload-labs xss-labs
  • 漏洞复现靶场 CVE 44228
  • 操作系统靶场 EG：vulnhub 靶场
  • CTF 靶场

第五部分

  1. 堡垒机 （安全产品 ）
  • 跳板机 jumpserver（规定同一个访问入口）
  • 运维审计系统 管理资源、审批审计、访问控制、事件记录
  1. WAF web 应用防火墙（Web Application Firewall）
    修复老旧软件老旧漏洞较为麻烦
    对 HTTP/HTTPS 的流量内容进行分析，拦截恶意攻击行为
    利用 waf 拦截外部攻击（命令前置的过滤拦截）
  • 硬件（厂商安装）
  • 云服务器保护
  • 软件（部署到 Apache Nginx 等 http server 中）
    【对应有绕过 waf】

  1. APT 攻击 （Advanced Persistent Threat）
    高级可持续威胁攻击
    指某些组织在网上对特定对象展开的持续有效的攻击活动
    报告：2022 深信服 APT 攻防趋势半年洞察

  2. 护网
    国家组织牵头 组织事业单位 国企单位 名企单位等展开攻防双方的网络安全演习（红队攻击 蓝队防御）

  3. CTF capture the flag 夺旗赛
    黑客技术竞赛（解题/攻防）
    方向 Reverse Pwn Crypto Misc Mobile
    在线 CTF： bugku 北京联合大学
    CTFHub bmzCTF 攻防世界 CTFSHOW

第六部分

  1. CVE 通用漏洞披露 （Common Vulnerablility and Exposures）
    Mitre
    例如： CVE-2021-44228
    http://www.cve.org/

  2. CNVD

  • 国家信息安全漏洞共享平台
  • 国家计算机应急响应中心 CNCERT 维护

  1. 应急响应
    组织为了应对各种安全时间所做的准备和事后采取的措施

  2. SRC
    security response center 企业的应急响应中心
    公益 SRC ： vulbox ..

  3. 网络空间测绘
    网络空间资源收录
    网络空间搜索引擎
    www.shodan.io
    fofa.io
    www.zoomeye.org

  4. ATT&CK
    （Adversarial Taactics，Techniques， and Common Konwledge）
    攻击者战术知识库
    Mitre
    风险分析模型 手机威胁情报 模拟 APT 攻击

  5. 逆向 reverse
    把程序还原成源代码，分析程序的运行过程
    （破解）

  6. DevOps Development+Operations
    开发 测试 运维 一体化

  7. CICD （实现 DecOps 理念）
    包括：

  • 持续集成 Continous Integration
  • 持续交付 Continous Delivery
  • 持续部署 Continous Deployment

具体技术：
Git 代码管理，Jenkins 版本管理，
代码扫描，自动化测试

  1. DevSecOps
    Development+Security+Operations
    安全开发和运维

  2. 等保（法律规定）
    网络安全登记保护
    要求相关行业的单位和公司的信息系统必须进行定级，在公安机关备案，建设整改，测评机构评级，并且持续维护和监督
