体系结构和参考模型(TCP)

TCP/IP 参考模型 与 五层模型

四层 TCP/IP 参考模型（事实模型）

（一）TCP/IP 协议栈

TCP/IP 协议栈是一组网络通信协议的集合，用于实现互联网上的数据传输和通信。它包含了一系列的协议层，每一层都负责不同的功能和任务，以确保有效的数据传输和通信。

主要协议层包括：

  1. 物理层（Physical Layer）：物理层负责定义传输介质、电气特性和机械连接等细节，例如以太网、光纤等。

  2. 数据链路层（Data Link Layer）：数据链路层提供了可靠的点对点数据传输，将原始的比特流划分为帧，并进行错误检测和纠正。常见的数据链路协议有以太网、Wi-Fi 等。

  3. 网络层（Network Layer）：网络层负责数据包的路由选择和转发，确定数据包如何从源节点传输到目标节点。最常见的网络层协议是 Internet 协议（IP），它使用 IP 地址来标识节点和网络。

  4. 传输层（Transport Layer）：传输层负责在应用程序之间提供端到端的可靠数据传输。其中最著名的协议是传输控制协议（TCP），它提供面向连接的可靠传输。此外，还有用户数据报协议（UDP），它提供无连接的不可靠传输。

  5. 应用层（Application Layer）：应用层是与最终用户交互的协议，包括 HTTP、FTP、SMTP、DNS 等。它们定义了数据的格式和交换规则，使应用程序能够通过网络进行通信。

理解

TCP/IP 协议栈提供了一个层次化的结构，每一层都依赖于下一层的服务，并向上层提供服务。通过这种方式，不同的设备和系统可以在互联网上进行通信，实现数据的可靠传输和应用之间的交互。

（二）四层 TCPIP 参考模型和 OSI 参考模型的相同点

（1）都分层

（2）基于独立的协议栈的概念

（3）都可以实现异构网络互联

（三）不同点

  • OSI 定义三点：接口 协议 服务

  • OSI 先出现，参考模型先于协议发明，不偏向特定协议

  • TCP/IP 设计之初就考虑到异构网互联问题，将 IP 作为重要层次

关于 IP 和无连接:
IP（Internet Protocol）是一种无连接协议。在 TCP/IP 协议栈中，IP 位于网络层，负责数据包的路由和转发。与传输控制协议（TCP）不同，IP 协议本身不建立连接或维护状态。它使用 IP 地址标识网络上的节点，并将数据包从源节点发送到目标节点。
由于 IP 是无连接的，它不提供可靠性保证。这意味着 IP 数据包在传输过程中可能会丢失、延迟或乱序。为了解决这些问题，通常会在 IP 之上使用 TCP 来提供可靠的数据传输服务。TCP 建立连接、进行流量控制、拥塞控制和重传等机制，以确保数据的可靠性和有序性。
另一方面，用户数据报协议（UDP）也是基于 IP 的传输协议，但它也是无连接的。UDP 提供了一种简单的、无状态的数据传输服务，适用于对可靠性要求不高、实时性较强的应用场景，如音频和视频流媒体、实时游戏等。
因此，IP 本身是一种无连接协议，而 TCP 和 UDP 则是在 IP 之上构建的传输协议，提供不同程度的可靠性和服务质量。
