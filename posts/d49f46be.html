学习 Burp Suite

一、Proxy 代理模块

（一）网络代理（HTTP 向）

1）作用

  • 突破 IP 限制
  • 隐藏 IP
  • 加速访问
  • …

2）Burp 代理的作用

拦截 HTTP（s）请求，并对请求和响应进行处理和利用

3）BP.Fiddler.WireShark 的区别

Burp Fiddler WireShark
平台 多平台 Windows 多平台
定位 渗透测试套件 Web 调试工具 网络封包分析工具
功能 渗透测试 调试 HTTP 请求 分析数据帧

4）浏览器代理

  1. 用操作系统设置范围影响太大
  2. 在浏览器内设置代理（目前是使用火狐浏览器代理）
  3. 插件：FoxyProxy

（二）BrupSuite 代理

1）启动操作

  1. Proxy → Intercept ON
  2. foxyproxy

2）内置操作

Forward 放行本次拦截的包，发送到服务器
Dropon/off 丢弃本次拦截的包
Intercept ON 拦截开关
Action 对数据的操作
Open in Browser 在浏览器打开这个 HTTP 请求

3）Options 详细设置

操作
Proxy Listeners 监听器
Intercept Client Requests 请求拦截规则
Intercept Server Response 响应拦截规则
Response ModificationMatch and Replace 响应结果修改匹配和替换
TLS Pass Through TLS 穿透
Miscellaneous 杂项

（三）证书安装

代理模式下访问 http://burp/ 安装证书到当前用户，省得报警。

二、Target 模块

（一）Target 模块的作用

1）和 HTTP ProxyHistory 的区别

  1. HTTP History 是按照时间顺序记录的
  2. Tarrget 是按照主机或域名分类记录的

2）模块作用

  1. 把握网站的整体情况
  2. 对一次工作的域进行分析（主域名，子域名，接口相同）
  3. 分析网站存在的攻击面（对一个软件系统可以采取得到攻击方法的集合，包括：字段，协议，接口，服务，硬件的攻击点）

（二）限定作用域范围

  1. 在 setting Scope 里面写入黑名单，白名单
  2. 黑名单（ExInclude）
  3. 白名单优先级比黑名单高（Include）

（三）站点地图的功能

SIte Map

1）自动（爬行）

2）手动（浏览器访问）

需要关闭 Proxy 从而获取全面的站点地图

（四）记录内容的操作

1）右键进入菜单后：

Add to scope 添加作用域
Engagement tools 交互工具
Compare site maps 站点比较
Expand branch 展开分支
Expand requested items 展开请求头目
Delete item 删除条目
Copy URLs in this branch 复制 URL
Copy links in this branch 复制链接
Copy as curl command 复制为 curl 命令
Save selected items 保存

2）Referer 字段

作用：告诉服务器当前请求是从哪个页面链接过来的
应用场景：

  1. 来源统计
  2. 防盗链

三、漏洞扫描模块

（一）整体介绍

1）扫描软件：

AWVS、Appscan、Nessus、Openvas、Goby、 Xray、ZAP…

2）23 版 BP 扫描

爬行 Crwal
审计 Audit（漏洞检测）

Scan（主动扫描） 给定地址，爬取内容，检测漏洞
Live task （被动扫描） 对经过 Proxy、Repeater、Intruder 的请求进行漏洞检测
live passive crawl from proxy(all traffic) 来自 Proxy 的被动流量抓
e audit from proxy(all traffic) 流量的实时审

4）主/被动扫描

Actively Scan：主动扫描 = Crawl and audit
Passively Scan：被动扫描 = Live audit

（二）主动扫描

  1. 方式：爬取所有链接，检测漏洞

  2. 特点：发送大量请求（影响服务器性能，不要主动扫描外站）

  3. 使用场合：开发、测试环境

  4. 针对漏洞：
    客户端的漏洞，如 XSS 、HTTP 头注入、操作重定向
    服务端的漏洞，如 SQL 注入、命令行注入、文件遍历

（三）被动扫描

  1. 方式：只检测经过 BP 代理服务器的地址，不爬取

  2. 特点：发送有限请求

  3. 使用场合：生产环境

  4. 针对漏洞：
    提交的密码为未加密的明文
    不安全的 cookie 的属性，例如缺少 HttpOnly 和安全标志
    cookie 的范围缺失。 跨域脚本包含和站点引用泄露
    表单值自动填充，尤其是密码。
    SSL 保护的内容缓存。 目录列表。
    提交密码后应答延迟。 session 令牌的不安全传输。
    敏感信息泄露，例如内部 IP 地址、电子邮件地址、堆枝跟踪等信息泄露。
    不安全的 ViewState 的配置 错误或不规范的 Content-Type 指令

（三）Scan 功能使用

1）Scan Configuration：爬行和审计的设置

爬行
Crwal Optimization 爬行的优化 最大链接深度 更快还是更完整
Crwal Limits 爬行最大限制 最大时间 最多链接 最大请求数
Login Functions 登录注册 登录操作：自动注册；用无效的用户名主动触发登录失败
Handling Application 错误处理 爬行过程中的错误处理，比如超时
Miscellaneous 杂项
审计
Audit Optimization 审计优化 扫描的速度和精确度
Issues Reported 问题报告 报告哪些漏洞：根据扫描类型或者漏洞类型 来过滤，默认全选
Handling Application Errors During Audit 审计过程出错 的处理 比如连接失败和传输超时默认：如果一个插 入点连续失败两次，就跳过，不再发送请求 （接口挂了） 如果连续两个插入点失败，跳过其他的插入 点（网站挂了）
Insertion Point Types 插入点的类型 URL 参数值、Body 里面的参数值、Cookie 值、 参数名字、HTTP 请求头、Body 完整内容、 URL 文件名、URL 目录
Modifying Parameter Locations 插入点位置 替换，交叉检测
gnored insertion Point 忽略的插入点
F r e q u e n t c l y O c c u r r i n g Insertion Points 插入点相同时 当大量的插入点结果没有区别的 时候，更加高效地扫描。 Mi s c Insertion Point Options 杂项 一个插入点的最大请求数量 J a v a S c r i p t Analysis JavaScript 审

四、Repeater 模块

用途：发起 HTTP 请求，分析响应；重放请求
把内容直接右键菜单传送到 repeater 模块就行

五、Intruder 模块

（一）作用和原理

1）作用：

对请求参数进行修改，分析响应内容，获得特征数据

2）本质：

  1. 自动化发起 HTTP 请求
  2. 基于现成字典或者生成字

3）用途

  1. 猜测用户名、密码等
  2. 寻找参数、目录等
  3. 枚举商品 ID、验证码
  4. 模糊测试（FUZZ）
    可替代工具： wfuzz（全部功能）、dirb（目录扫描）、hydra（暴 破）…

（二）实现暴力破解

1）流程

  1. 从其他模块发送或者手动填写
  2. 选择攻击模式 Attack type
  3. 选择攻击字段 Positions 、设置 payload
  4. 其他设置（线程池等）
  5. 发起攻击
  6. 查看结果

2）payload type

Simple list 简单字典 添加、粘贴或者从文件读取字典，或者使用预定义的字典
Runtime file 运行时文件 运行时，Burp Intruder 将读取文件的每一行作为一个 Payload
Custom iterator 自定义迭代器 这个是占位填充的一种方式，最多 8 位
Character substitution 字符替换 把字典里面相应的字符进行替换
Casemodification 大小写修改 要不要保持原样的，要不要全部大写的，要不要全小写的， 要不要驼峰命名的
Reciusvive grep 递归查找 用来提取相应数据的比如拿到 PHPSESSIONID，拿到 TOKEN 等等，可以通过格式匹配抓取到对应的字段值
Illegal unicode 非法 Unicode 编码 用于绕过正则表达式的过滤验证
Character blocks 字符块 比如生成 100A，200 个+号，300 个数字 1 等等 numbers 数字组合 dates 日期组合
Brute forcer 暴力破解 暴力枚举，最后一位先固定，然后一个个改
Null payloads 空 payload 不需要设置 payload
Character frobber 字符 frobber 依次修改指定字符串在每个字符位置的值，每次在原字符上递增一个该字符的 ASCII 码。
Bit flipper Bit 翻转 对预设的 Payload 原始值，按照比特位，依次进行修改
Username generator 用户名生成器 主要用于用户名和 email 帐号的自动生成
ECBblock shuffler ECB 加密块洗 牌 基于 ECB 加密模式的 Payload 生成器
Extension - generated BPayload 生成插件 基于 Burp 插件来生成 Payload 值，需要安装插件
Copyother payload Payload 复制 是将其他位置的参数复制到 Payload 位置上（比如密码要输 入两遍）

（三）三种其他的爆破模式

1）Battering ram 攻城锤

所有字段的值相同，来自同一个字典

2）Pitchfork 草叉

有字段的值相同，来自同一个字典，按顺序一一对应

3）Cluster bomb 榴霰弹

所有字典全部交叉验证

（四）标记结果

在 Grep-March 中 Add 需要标记的内容

（五）获得 CSRF Token（类似令牌）

凭借令牌依次轮流访问（表现为只能单线程发包）
Pitchfork
放字典
payloadset payloadtype 选择 recursive grep
payloadoptions add fetch response（记得把第一次的令牌复制好）

（六）抓取响应包

在抓包页面右键 入 DoIntercept Response to this respect
